griot_core.dataframe_validation¶

Griot DataFrame Validation - Pandera-Based Validation Engine

This module provides DataFrame validation using Pandera as the underlying validation engine. It supports multiple DataFrame types: - pandas DataFrames - polars DataFrames - PySpark DataFrames - Dask DataFrames

The validation engine uses a registry pattern to support different DataFrame types with their respective validators.

Installation (composable - install only what you need):

pip install griot-core[pandas] # pandas support pip install griot-core[polars] # polars support pip install griot-core[spark] # PySpark support pip install griot-core[dask] # Dask support pip install griot-core[pandas,polars] # multiple types pip install griot-core[all] # all DataFrame types

class griot_core.dataframe_validation.PanderaSchemaGenerator[source]¶

Bases: object

Generate Pandera schemas from Schema with ODCS quality rules.

Converts Schema field definitions to Pandera column schemas, transforming ODCS quality rules into Pandera checks.

Supported ODCS quality types:

  • library: Predefined metrics (nullValues, missingValues, etc.)

  • text: Human-readable only (skipped)

  • sql: SQL queries (skipped - requires DB context)

  • custom: Vendor-specific (skipped)

Supported library metrics:

  • nullValues: Count of null values

  • missingValues: Count of missing values (null, ‘’, ‘N/A’, etc.)

  • invalidValues: Count of values not matching pattern/valid set

  • duplicateValues: Count of duplicate values

Example

generator = PanderaSchemaGenerator() pandera_schema = generator.generate(griot_schema)

TYPE_MAP: dict[str, str] = {'array': 'object', 'boolean': 'bool', 'date': 'datetime64[ns]', 'datetime': 'datetime64[ns]', 'float': 'float64', 'integer': 'int64', 'object': 'object', 'string': 'str'}¶

griot_core.dataframe_validation.validate_schema_data(schema, data, *, mode=ValidationMode.LAZY)[source]¶

Validate data against a single Schema.

This is the primary validation function for schema-level data validation. It automatically detects the data type (DataFrame or list of dicts) and uses the appropriate validator.

Parameters:

  • schema (Schema) – Schema to validate against

  • data (Any) – Data to validate (DataFrame or list of dicts)

  • mode (ValidationMode) – Validation mode (LAZY collects all errors, EAGER fails fast)

Returns:

DataFrameValidationResult with validation details

Return type:

DataFrameValidationResult

Example

from griot_core import Schema, validate_schema_data

schema = Schema.from_dict(schema_dict)

# Validate pandas DataFrame result = validate_schema_data(schema, df)

# Validate list of dicts result = validate_schema_data(schema, [{“id”: “1”}, {“id”: “2”}])

if not result.is_valid:
for error in result.errors:

print(error)

Example

mapping = {

employees_schema: employees_df, departments_schema: departments_df,

}
