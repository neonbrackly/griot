Types API¶

The types module provides enumerations and type definitions for Griot Core.

Enums¶

ContractStatus¶

class griot_core.ContractStatus¶

Contract lifecycle status based on ODCS specification.

DRAFT = "draft"¶

Contract is under development, not for production use.

ACTIVE = "active"¶

Contract is production-ready, consumers can depend on it.

DEPRECATED = "deprecated"¶

Contract is being phased out, consumers should migrate.

RETIRED = "retired"¶

Contract is no longer available.

Example

from griot_core import Contract, ContractStatus

# Create a draft contract
contract = Contract(
    id="my-contract",
    status=ContractStatus.DRAFT
)

# Promote to active
contract.status = ContractStatus.ACTIVE

# Mark for deprecation
contract.status = ContractStatus.DEPRECATED

Any type.

Severity¶

class griot_core.Severity¶

Error/issue severity levels.

ERROR = "error"¶

Critical issue that must be fixed.

WARNING = "warning"¶

Potential issue that should be reviewed.

INFO = "info"¶

Informational message.

Example

from griot_core import lint_contract, Severity

issues = lint_contract(contract)
errors = [i for i in issues if i.severity == Severity.ERROR]
warnings = [i for i in issues if i.severity == Severity.WARNING]

QualityMetric¶

class griot_core.QualityMetric¶

ODCS quality metrics for library-type checks.

Property-Level Metrics (operate on a single column):

NULL_VALUES = "nullValues"¶

Count of null/None values in a column.

MISSING_VALUES = "missingValues"¶

Count of values considered missing (null, empty string, N/A, etc.).

INVALID_VALUES = "invalidValues"¶

Count of values failing validation rules.

DUPLICATE_VALUES = "duplicateValues"¶

Count of duplicate values in a column.

Schema-Level Metrics (operate on entire dataset):

ROW_COUNT = "rowCount"¶

Total number of rows in the dataset.

DUPLICATE_ROWS = "duplicateRows"¶

Count of duplicate rows across specified columns.

Properties

is_property_level: bool¶

True if this metric operates at the property/column level.

is_schema_level: bool¶

True if this metric operates at the schema/table level.

Example

from griot_core import QualityMetric

metric = QualityMetric.NULL_VALUES
print(metric.is_property_level)  # True
print(metric.is_schema_level)    # False

metric = QualityMetric.ROW_COUNT
print(metric.is_property_level)  # False
print(metric.is_schema_level)    # True

QualityOperator¶

class griot_core.QualityOperator¶

ODCS comparison operators for quality rule thresholds.

MUST_BE = "mustBe"¶

Equals (==).

MUST_NOT_BE = "mustNotBe"¶

Not equals (!=).

MUST_BE_GREATER_THAN = "mustBeGreaterThan"¶

Greater than (>).

MUST_BE_GREATER_OR_EQUAL_TO = "mustBeGreaterOrEqualTo"¶

Greater than or equal (>=).

MUST_BE_LESS_THAN = "mustBeLessThan"¶

Less than (<).

MUST_BE_LESS_OR_EQUAL_TO = "mustBeLessOrEqualTo"¶

Less than or equal (<=).

MUST_BE_BETWEEN = "mustBeBetween"¶

Between two values (exclusive).

MUST_NOT_BE_BETWEEN = "mustNotBeBetween"¶

Not between two values.

Properties

comparison_type: str¶

Internal comparison type code (eq, ne, gt, ge, lt, le, between, not_between).

Methods

compare(value: float, threshold: Any) → bool¶

Compare a metric value against a threshold.

Parameters:

  • value – The calculated metric value

  • threshold – The comparison threshold (number or list for between)

Returns:

True if comparison passes

op = QualityOperator.MUST_BE_LESS_THAN
op.compare(5, 10)   # True (5 < 10)
op.compare(15, 10)  # False (15 < 10)

op = QualityOperator.MUST_BE_BETWEEN
op.compare(5, [1, 10])  # True (1 < 5 < 10)

QualityRule Builder¶

class griot_core.QualityRule¶

Builder class for creating ODCS-compliant quality rules.

Provides type-safe factory methods to build quality rules using enums instead of raw strings, reducing errors when defining contracts.

Property-Level Rules (for field quality):

classmethod null_values(*, must_be=None, must_not_be=None, must_be_less_than=None, must_be_less_or_equal_to=None, must_be_greater_than=None, must_be_greater_or_equal_to=None, must_be_between=None, unit=QualityUnit.ROWS, rule_id=None, name=None) → dict¶

Create a null values quality rule.

Parameters:

  • must_be – Exact count of null values allowed

  • must_not_be – Count that null values must not equal

  • must_be_less_than – Maximum null values (exclusive)

  • must_be_less_or_equal_to – Maximum null values (inclusive)

  • must_be_greater_than – Minimum null values (exclusive)

  • must_be_greater_or_equal_to – Minimum null values (inclusive)

  • must_be_between – Range [min, max] for null values

  • unit – ROWS for absolute count, PERCENT for percentage

  • rule_id – Unique identifier for the rule

  • name – Human-readable name

Returns:

ODCS-compliant quality rule dictionary

# No nulls allowed
QualityRule.null_values(must_be=0)
# {'metric': 'nullValues', 'mustBe': 0}

# Less than 5% nulls
QualityRule.null_values(
    must_be_less_than=5,
    unit=QualityUnit.PERCENT
)
# {'metric': 'nullValues', 'mustBeLessThan': 5, 'unit': 'percent'}
classmethod missing_values(*, must_be=None, must_not_be=None, must_be_less_than=None, must_be_less_or_equal_to=None, must_be_greater_than=None, must_be_greater_or_equal_to=None, must_be_between=None, unit=QualityUnit.ROWS, missing_values_list=None, rule_id=None, name=None) → dict¶

Create a missing values quality rule.

Parameters:

missing_values_list – Custom list of values considered missing

Returns:

ODCS-compliant quality rule dictionary

# No missing values (default: null, empty, N/A)
QualityRule.missing_values(must_be=0)

# Custom missing values
QualityRule.missing_values(
    must_be=0,
    missing_values_list=[None, '', 'N/A', 'NULL', '-']
)
classmethod invalid_values(*, must_be=None, must_not_be=None, must_be_less_than=None, must_be_less_or_equal_to=None, unit=QualityUnit.ROWS, valid_values=None, pattern=None, min_value=None, max_value=None, min_length=None, max_length=None, rule_id=None, name=None) → dict¶

Create an invalid values quality rule.

Parameters:

  • valid_values – List of allowed values (enum constraint)

  • pattern – Regex pattern values must match

  • min_value – Minimum numeric value allowed

  • max_value – Maximum numeric value allowed

  • min_length – Minimum string length

  • max_length – Maximum string length

Returns:

ODCS-compliant quality rule dictionary

# Enum validation
QualityRule.invalid_values(
    must_be=0,
    valid_values=['active', 'inactive', 'pending']
)

# Pattern validation
QualityRule.invalid_values(
    must_be=0,
    pattern=r'^[\w.-]+@[\w.-]+\.\w+$'
)

# Range validation
QualityRule.invalid_values(
    must_be=0,
    min_value=0,
    max_value=150
)

# Length validation
QualityRule.invalid_values(
    must_be=0,
    min_length=3,
    max_length=50
)
classmethod duplicate_values(*, must_be=None, must_not_be=None, must_be_less_than=None, must_be_less_or_equal_to=None, unit=QualityUnit.ROWS, rule_id=None, name=None) → dict¶

Create a duplicate values quality rule for a single column.

# No duplicates (unique values)
QualityRule.duplicate_values(must_be=0)

Schema-Level Rules (for schema quality):

classmethod row_count(*, must_be=None, must_not_be=None, must_be_greater_than=None, must_be_greater_or_equal_to=None, must_be_less_than=None, must_be_less_or_equal_to=None, must_be_between=None, rule_id=None, name=None) → dict¶

Create a row count quality rule (schema-level).

# At least 1 row
QualityRule.row_count(must_be_greater_than=0)

# Exactly 1000 rows
QualityRule.row_count(must_be=1000)

# Between 100 and 10000 rows
QualityRule.row_count(must_be_between=[100, 10000])
classmethod duplicate_rows(*, must_be=None, must_not_be=None, must_be_less_than=None, must_be_less_or_equal_to=None, unit=QualityUnit.ROWS, properties=None, rule_id=None, name=None) → dict¶

Create a duplicate rows quality rule (schema-level).

Parameters:

properties – List of column names to check. If None, checks all columns.

# No duplicate rows on composite key
QualityRule.duplicate_rows(
    must_be=0,
    properties=['order_id', 'line_number']
)

# No completely duplicate rows
QualityRule.duplicate_rows(must_be=0)

Complete Example

from griot_core import Schema, Field, QualityRule, QualityUnit

class CustomerSchema(Schema):
    # Schema-level quality rules
    _quality = [
        QualityRule.row_count(must_be_greater_than=0),
        QualityRule.duplicate_rows(must_be=0, properties=['customer_id']),
    ]

    customer_id: str = Field(
        "Customer ID",
        primary_key=True,
        quality=[
            QualityRule.null_values(must_be=0),
            QualityRule.duplicate_values(must_be=0),
            QualityRule.invalid_values(must_be=0, pattern=r'^CUST-\d{6}$'),
        ]
    )

    email: str = Field(
        "Email address",
        unique=True,
        quality=[
            QualityRule.null_values(must_be=0),
            QualityRule.invalid_values(
                must_be=0,
                pattern=r'^[\w.-]+@[\w.-]+\.\w+$'
            ),
        ]
    )

    age: int = Field(
        "Customer age",
        nullable=True,
        quality=[
            QualityRule.null_values(
                must_be_less_than=10,
                unit=QualityUnit.PERCENT
            ),
            QualityRule.invalid_values(
                must_be=0,
                min_value=0,
                max_value=150
            ),
        ]
    )

Validation Types¶

FieldValidationError¶

class griot_core.FieldValidationError¶

Single field validation error.

field: str¶

Field name that failed validation.

row: int | None¶

Row number (0-indexed) or None for column-level errors.

value: Any¶

The value that failed validation.

constraint: str¶

The constraint that was violated.

message: str¶

Human-readable error message.

severity: Severity¶

Error severity (default: ERROR).

Methods

to_dict() → dict¶

Convert to dictionary representation.

Example

error = FieldValidationError(
    field="email",
    row=5,
    value="invalid-email",
    constraint="pattern",
    message="Value does not match email pattern"
)

print(error)  # "email[row 5]: Value does not match email pattern"
print(error.to_dict())
