API Reference¶

This page provides detailed API documentation for griot-enforce.

RuntimeValidator¶

class RuntimeValidator(registry_url=None, api_key=None, cache_ttl=300, report_results=True)¶

Core validator for runtime data validation.

Wraps griot-core SDK validation with runtime features including registry integration, contract caching, and result reporting.

Parameters:

  • registry_url (str | None) – Registry URL. Can also be set via GRIOT_REGISTRY_URL env var.

  • api_key (str | None) – API key for registry. Can also be set via GRIOT_API_KEY env var.

  • cache_ttl (int) – Contract cache TTL in seconds. Default: 300 (5 minutes).

  • report_results (bool) – Whether to report validation results to registry. Default: True.

Example:

validator = RuntimeValidator(
    registry_url="https://registry.example.com",
    cache_ttl=600,  # 10 minute cache
)

Methods¶

RuntimeValidator.validate(contract_id, data, version=None, fail_on_error=True, verify_masking=False, environment=None)¶

Validate data against a contract from the registry.

Parameters:

  • contract_id (str) – Contract ID in the registry.

  • data (pd.DataFrame | list[dict]) – Data to validate. Can be a pandas DataFrame or list of dicts.

  • version (str | None) – Specific contract version. Default: latest.

  • fail_on_error (bool) – Raise RuntimeValidationError on validation failure. Default: True.

  • verify_masking (bool) – Also verify PII masking compliance (FR-ENF-009). Default: False.

  • environment (str | None) – Environment name for masking checks (e.g., “staging”, “dev”).

Returns:

ValidationResult from griot-core.

Return type:

ValidationResult

Raises:

  • RuntimeValidationError – If validation fails and fail_on_error=True.

  • MaskingViolationError – If masking verification fails and fail_on_error=True.

Example:

result = validator.validate(
    "customer-profile",
    dataframe,
    verify_masking=True,
    environment="staging",
)

RuntimeValidator.check_residency(contract_id, region=None, destination=None, version=None, fail_on_violation=False)¶

Check if data can be stored in a given region (FR-ENF-008).

Auto-detects region from cloud URIs including AWS S3, Azure Blob, and GCP GCS.

Parameters:

  • contract_id (str) – Contract ID in the registry.

  • region (str | None) – Explicit target region for data storage.

  • destination (str | None) – Cloud URI to auto-detect region from (e.g., s3://bucket-eu-west-1/).

  • version (str | None) – Specific contract version.

  • fail_on_violation (bool) – Raise ResidencyViolationError if not compliant. Default: False.

Returns:

Dictionary with compliance status and violations.

Return type:

dict[str, Any]

Raises:

  • ResidencyViolationError – If not compliant and fail_on_violation=True.

  • ValueError – If neither region nor destination is provided.

Supported URI Patterns:

  • AWS S3: s3://bucket-eu-west-1/path or s3://bucket.s3.eu-west-1.amazonaws.com/

  • Azure Blob: https://account.blob.core.windows.net/container

  • GCP GCS: gs://bucket-us-central1/path

Example:

# Auto-detect from S3 URI
result = validator.check_residency(
    "customer-profile",
    destination="s3://my-bucket-eu-west-1/data/",
    fail_on_violation=True,
)

# Response includes:
# {
#     "compliant": True,
#     "detected_region": "eu-west-1",
#     "destination": "s3://my-bucket-eu-west-1/data/",
#     "target_region": "eu-west-1",
#     "allowed_regions": ["eu-west-1", "eu-central-1"],
# }
RuntimeValidator.verify_masking(contract_id, data, version=None, environment=None, fail_on_violation=False)¶

Verify that PII fields are properly masked (FR-ENF-009).

Parameters:

  • contract_id (str) – Contract ID in the registry.

  • data (pd.DataFrame | list[dict]) – Data to check for masking compliance.

  • version (str | None) – Specific contract version.

  • environment (str | None) – Environment name. Skips verification in “prod”/”production”.

  • fail_on_violation (bool) – Raise MaskingViolationError if not compliant. Default: False.

Returns:

Dictionary with masking verification results.

Return type:

dict[str, Any]

Raises:

MaskingViolationError – If not compliant and fail_on_violation=True.

Masking Strategies Checked:

  • redact: Expects [REDACTED] in value

  • partial: Expects * characters (e.g., ****1234)

  • hash: Expects 32+ character hex string

Example:

result = validator.verify_masking(
    "customer-profile",
    dataframe,
    environment="staging",
)

# Response:
# {
#     "compliant": False,
#     "environment": "staging",
#     "pii_fields_checked": ["email", "phone", "ssn"],
#     "violations": [
#         {
#             "field": "email",
#             "expected_strategy": "partial",
#             "message": "Field 'email' does not appear to be properly masked",
#             "pii_category": "contact_info",
#         }
#     ],
# }

Exceptions¶

exception RuntimeValidationError(message, contract_id=None, error_count=0, error_rate=0.0)¶

Raised when runtime validation fails.

Parameters:

  • message (str) – Error message.

  • contract_id (str | None) – Contract ID that failed validation.

  • error_count (int) – Number of validation errors.

  • error_rate (float) – Error rate (errors / total rows).

Attributes:

  • contract_id (str | None): Contract ID

  • error_count (int): Number of errors

  • error_rate (float): Error rate as decimal

exception ResidencyViolationError(message, contract_id=None, violations=None, detected_region=None, allowed_regions=None)¶

Raised when data residency requirements are violated.

Parameters:

  • message (str) – Error message.

  • contract_id (str | None) – Contract ID.

  • violations (list[dict] | None) – List of violation details.

  • detected_region (str | None) – Region detected from URI.

  • allowed_regions (list[str] | None) – List of allowed regions.

Attributes:

  • contract_id (str | None): Contract ID

  • violations (list[dict]): Violation details

  • detected_region (str | None): Detected region

  • allowed_regions (list[str]): Allowed regions

exception MaskingViolationError(message, contract_id=None, violations=None)¶

Raised when PII masking requirements are violated.

Parameters:

  • message (str) – Error message.

  • contract_id (str | None) – Contract ID.

  • violations (list[dict] | None) – List of masking violations.

Attributes:

  • contract_id (str | None): Contract ID

  • violations (list[dict]): List of violations with field, expected_strategy, message
